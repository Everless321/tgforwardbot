# Fix handler registration causing duplicate forwarding

## Task
Fix event handler lifecycle - old handlers not removed, toggle ON/OFF not syncing to runtime.

## Changes
- `handlers.py`: Track current handler via `_current_handler`, remove old before registering new
- `rules.py`: Extract `_rebuild_handlers()` - reload all enabled rules from DB and re-register
- `rules.py`: create/update/delete all call `_rebuild_handlers()` for immediate effect

## Result
- [x] No more duplicate forwarding when rules change
- [x] Toggle ON/OFF takes effect immediately (no restart needed)
- [x] Delete rule stops forwarding immediately
- [x] Add rule starts forwarding immediately
